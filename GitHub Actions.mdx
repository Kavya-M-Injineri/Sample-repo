#  GitHub Actions – Complete Guide

##  Definition

**GitHub Actions** is a **CI/CD (Continuous Integration and Continuous Deployment)** tool that allows developers to **automate build, test, and deployment workflows directly from a GitHub repository**.

 In simple terms:  
It helps you automate tasks like compiling code, running tests, and deploying applications whenever changes are made.

---

##  Why GitHub Actions is Important

-  Saves time by automating repetitive tasks  
-  Detects bugs early through automated testing  
-  Ensures code quality  
-  Enables continuous delivery and deployment  
-  Fully integrated with GitHub  

---

##  Key Concepts

### 1.  Workflow
A **workflow** is an automated process defined in a YAML file.

 Location:

---

### 2.  Event
An **event** triggers the workflow.

Examples:
- `push` → when code is pushed  
- `pull_request` → when a PR is created  
- `schedule` → run at specific times  

---

### 3.  Job
A **job** is a set of steps executed on the same runner.

- A workflow can contain multiple jobs  
- Jobs run in parallel by default  

---

### 4.  Step
A **step** is an individual task within a job.

Examples:
- Install dependencies  
- Run Maven build  
- Execute tests  

---

### 5.  Runner
A **runner** is the machine that executes the workflow.

Types:
- GitHub-hosted (e.g., `ubuntu-latest`)  
- Self-hosted  

---

### 6.  Action
An **action** is a reusable component used in workflows.

Example:
```yaml
uses: actions/checkout@v3

name: Java CI with Maven

on:
  push:
    branches: [ "main" ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v3

      - name: Set up Java
        uses: actions/setup-java@v3
        with:
          java-version: '17'

      - name: Build project
        run: mvn clean install
